Grid-Cell Neighborhoods Visualization

This visualization demonstrates solutions to the Grid-Cell Neighborhoods problem, which counts cells within Manhattan distance N of any positive value in a 2D grid. Current grid size:

Example 1: One positive cell fully contained; N=3

Total cells in neighborhood: 0 (Expected: 25)
In this example, there is only one positive value, located near the center of a grid large enough to contain the N=3 neighborhood comfortably.

Example 2: One positive cell near an edge; N=3

Total cells in neighborhood: 0 (Expected: 21)
This is similar to Example 1, except that the positive cell is less than 3 steps from the left-hand side of the array. Four of the cells from Example 1 have "fallen off" the left-hand edge of the array.

Example 3: Two positive values with disjoint neighborhoods; N=2

Total cells in neighborhood: 0 (Expected: 26)
Each of the positive values has a 13-cell neighborhood, so the total number of cells within N=2 steps of a positive value is 26.

Example 4: Two positive values with overlapping neighborhoods; N=2

Total cells in neighborhood: 0 (Expected: 21)
The cells that are members of more than one neighborhood are only counted once. There are 5 cells that are members of both neighborhoods independently.
